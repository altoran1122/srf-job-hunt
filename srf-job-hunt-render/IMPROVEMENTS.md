# SRF Job Hunt 개선 메모

## 다음에 한 번에 고칠 항목

1. 자동 태그 정확도 개선
   - 현재 IB/WM 등 직무 태그는 공고 제목, 회사명, JD 텍스트의 키워드 기반으로 자동 추정한다.
   - `wealth` 같은 단어만으로 WM 태그가 붙는 문제가 있다. 예: `sovereign wealth fund`는 WM/PB가 아니므로 제외해야 한다.
   - WM은 `wealth management`, `private banking`, `PB`, `고객자산관리`, `프라이빗뱅커`처럼 더 명확한 표현 위주로 판별한다.
   - IB도 `ib` 단순 포함이 아니라 단어 경계와 실제 표현을 기준으로 판별한다.
   - 자동 태그는 가능하면 `자동분류` 또는 `추정` 성격이 드러나게 하고, 관리자가 확정 분류할 수 있는 구조를 검토한다.

2. 추천 공고 기준 변경
   - 웹페이지가 임의로 추천을 정하는 방식 대신, 사용자 반응 기반으로 추천 공고를 정한다.
   - 한 공고에 대해 `관심` 또는 `지원완료`를 누른 지원자가 합산 5명 이상이면 추천 공고로 분류한다.
   - 같은 사용자가 `관심`과 `지원완료`를 모두 눌러도 추천 카운트는 1명으로 계산한다.
   - 사용자가 관심/지원완료를 취소하면 추천 카운트에서도 빠져야 한다.
   - 추천 공고는 카드에서 시각적으로 더 눈에 띄게 표시한다.
   - 예시: 더 짙은 테두리, 옅은 배경색, 추천 배지 등.

3. 슈퍼루키 `주니어경력` 제외
   - 수집 대상은 `신입`, `주니어`, `인턴`으로 제한한다.
   - 슈퍼루키의 `주니어경력`은 우리가 원하는 인턴/신입급 공고가 아니므로 자동 수집에서 제외한다.
   - 이미 저장된 `주니어경력` 공고는 다음 정리 작업 때 숨김 처리하거나 삭제 대상으로 검토한다.

4. 금융회사 비금융직렬 제외
   - 비금융회사의 금융직렬은 수집 대상에 포함한다. 예: 자동차 금융사/핀테크/일반 기업의 재무, 리스크, 투자, 전략금융 직무.
   - 금융회사의 비금융직렬은 수집 대상에서 제외한다. 예: 금융회사 CEO Executive Assistant, 총무, 일반 IT 운영, 시설/운영성 직무.
   - 금융 관련성은 회사 업종만으로 판단하지 말고, 직무/JD 텍스트를 중심으로 판별한다.
   - 회사가 금융권이어도 직무가 금융 커리어와 직접 연결되지 않으면 `비대상` 또는 `숨김`으로 처리한다.

5. 오래된 공고 자동 숨김
   - 상시채용처럼 비정기적으로 계속 intake를 받는 직렬이 아니라면, 공고 게시일 또는 수집일 기준 3개월 후에는 기본 목록에서 숨긴다.
   - 마감일이 명확히 지난 공고도 기본 목록에서 숨기는 방향을 검토한다.
   - 숨김 처리된 공고는 관리자 화면이나 별도 필터에서만 확인할 수 있게 한다.

6. 추천 카운트를 위한 사용자 식별
   - `관심` 또는 `지원완료`를 누른 사람이 5명 이상인지 세려면, 최소한의 사용자 식별이 필요하다.
   - 계정 없이 브라우저 localStorage만 쓰면 같은 사람이 여러 기기에서 여러 명처럼 잡히거나, 브라우저 데이터를 지우면 카운트가 흔들린다.
   - 권장안: SRF 공용 비밀번호로 사이트에 들어오고, 첫 사용 시 이름/닉네임을 선택하거나 입력해 개인 프로필을 만든다.
   - 더 정확한 안: 관리자가 미리 멤버 계정을 만들고, 각 멤버가 자기 계정으로 로그인한다.
   - 추천 여부는 저장된 사용자별 상태를 기준으로 서버에서 동적으로 계산한다.
